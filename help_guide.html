User Guide

Track View

The Track View is your main interface for browsing, managing, and interacting with your music library. It displays all tracks in a searchable, sortable table with various customization options.

Key Features

1. Table Display

  • Shows all tracks from your music library

  • Sortable by clicking column headers

2. Column Management

  • Columns are grouped by category (General, Technical, etc.)

  • Click "Toggle Columns" button to show/hide specific columns

    • Columns in "Technical" category are hidden by default

3. Search Functionality

  • Type in the search bar to filter tracks

  • Searches across ALL columns (artist name, album, title, etc.)

  • Case-insensitive searching

  • Real-time filtering as you type

4. Track Selection & Actions

  • Single Click: Select a track

  • Ctrl+Click / Shift+Click: Select multiple tracks

  • Double Click: Play the selected track immediately

  • Delete Key: Delete selected track(s) from database or disk

Drag & Drop Support

  • Select multiple tracks and drag them to playlist areas

  • Visual feedback shows number of tracks being dragged

Overview

The Influence Graph View is an interactive visualization tool that displays relationships between artists using a force-directed graph layout. It helps users understand influence networks, identify key influencers, and explore connections within the artistic community.

Key Features

1. Graph Visualization

  • Force-Directed Layout: Nodes automatically arrange themselves based on:

    • Repulsion between nodes (keeps them separated)

    • Attraction along edges (pulls connected nodes together)

    • Community clustering (groups related artists)

  • Directional Arrows: Shows influence direction (influencer → influenced)

  • Node Sizing: Larger nodes represent more influential artists

  • Color Coding: Nodes are grouped by detected communities (Louvain algorithm)

2. Navigation Controls

  • Zoom: Use mouse wheel to zoom in/out

  • Pan: Click and drag to move around the canvas

  • Infinite Canvas: No boundaries - explore freely

  • Auto-Settle: Graph automatically stabilizes when movement stops

3. Interactive Elements

  • Artist Nodes: Click to select (functionality may be implemented elsewhere)

  • Hover Effects: Visual feedback on nodes and edges

  • Dynamic Updates: Graph updates in real-time when adding/removing relationships

Interface Components

Main Controls (Top Section)

  • Refresh Graph Button: Recalculates and redraws the entire graph

  • Add Influence Button: Opens dialog to create new influence relationships

  • Remove Influence Button: Opens dialog to delete existing relationships

Graph Display Area

  • Nodes: Represent individual artists

  • Edges: Represent influence relationships

  • Arrows: Indicate direction of influence (placed at midpoint of edges)

How to Use

1. Viewing the Graph

  • The graph automatically loads when you open the view

  • Artists with more influence connections appear larger

  • Closely connected artists form visible clusters

  • Use pan and zoom to explore different areas

2. Adding New Influences

  1. Click "Add Influence" button

  2. Select or create artists in the dialog

  3. Specify who influenced whom

  4. The graph updates incrementally without full refresh

3. Removing Influences

  1. Click "Remove Influence" button

  2. Select a relationship from the list

  3. Confirm removal

  4. Graph refreshes to show updated relationships

4. Understanding the Layout

  • Tight Clusters: Artists with strong mutual connections

  • Central Nodes: Highly connected/influential artists

  • Peripheral Nodes: Less connected artists

  • Long Edges: Less direct/strong influence relationships

Interpretation Guide

Node Size

  • Large Nodes: Artists who influenced many others (high descendant count)

  • Small Nodes: Artists with fewer direct influences

Node Positioning

  • Center of Clusters: Key influencers within communities

  • Between Clusters: Artists bridging different communities

  • Isolated Nodes: Artists with unique or limited connections

Edge Meaning

  • Thick/Short Edges: Strong/direct influence relationships

  • Thin/Long Edges: Weaker/indirect relationships

  • Arrow Position: Always at midpoint, not necessarily near target

Community Colors

  • Same color indicates artists in the same detected community

  • Different colors represent distinct artistic circles or movements

Tips for Best Experience

  1. Let Graph Settle: Allow a few seconds for the force layout to stabilize

  2. Zoom for Detail: Zoom in to see artist names clearly

  3. Pan to Explore: The canvas is infinite - don't hesitate to move around

  4. Refresh After Major Changes: Use refresh if the layout becomes unclear

  5. Check Logs: Technical details and warnings appear in the application logs

Technical Notes

Data Requirements

  • Requires at least one influence relationship to display

  • Only shows artists involved in influence relationships

  • Automatically excludes isolated artists (no connections)

Performance

  • Optimized for up to hundreds of nodes

  • Force layout stops automatically when stable

  • Incremental updates minimize computation for small changes

Algorithms Used

  • ForceAtlas2-inspired layout: For node positioning

  • Louvain Community Detection: For grouping similar artists

  • PageRank Decay: For influence scoring

  • Descendant Counting: For node sizing

Troubleshooting

Common Issues

  1. No Graph Displayed

    • Check if influence relationships exist in database

    • Verify database connection

    • Click "Refresh Graph"

  2. Graph Not Stabilizing

    • Wait a few more seconds

    • Click "Refresh Graph" to restart layout

    • Check console for errors

  3. Names Not Visible

    • Zoom in closer

    • Check if artist names are in database

  4. Missing Relationships

    • Verify relationships were saved to database

    • Use "Refresh Graph" to reload all data

Error Messages

  • "No artists with influence relationships found": Add some relationships first

  • "Failed to display global graph": Check logs for detailed error

  • "Partial Error": Some but not all updates applied; refresh to sync

Advanced Features

Community Detection

  • Automatically groups artists based on connection patterns

  • Useful for identifying artistic movements or schools

Influence Scoring

  • Combines direct descendant count with PageRank algorithm

  • Accounts for both quantity and quality of influence

Dynamic Updates

  • New artists added incrementally

  • Edge updates without full recomputation

  • Mass recalibration for force layout

Best Practices for Data Entry

  1. Complete Relationships: Enter both sides of influence when known

  2. Verify Names: Ensure artist names are consistent

  3. Start Small: Begin with key relationships, then expand

  4. Regular Maintenance: Remove outdated or incorrect relationships

Support

For persistent issues:

  1. Check the application logs

  2. Verify database integrity

  3. Contact system administrator

  4. Report bugs with specific reproduction steps

Note: This guide assumes standard installation and configuration. Some features may vary based on specific implementation.



  • Edit all editable track fields

  • Modify artist roles, genres, and place associations

  • Save changes to update the database

Multi-Track Edit:

  • Edit common fields across multiple tracks simultaneously

  • Only fields marked as "multiple-editable" are available

  • Useful for batch updates (e.g., changing genre for multiple tracks)

Delete Functionality

Press Delete key or use context menu to delete tracks:

Two Delete Options:

  1. Delete from database only: Keeps audio files on disk

  2. Delete from disk and database: Permanently removes files

Safety Features:

  • Confirmation dialog before any deletion

  • Additional warning for permanent file deletion

Keyboard Shortcuts

  • Delete: Delete selected track(s)

  • Double-click: Play selected track

  • Arrow Keys: Navigate between tracks

  • Ctrl+A: Select all tracks (when table has focus)

Places View

Overview

The Places Manager is a dual-view interface for managing geographic locations linked to your music library. It allows you to visualize places on an interactive map and organize them in a hierarchical list view.

Key Features

1. Dual View Interface

  • Map View: Interactive map with color-coded markers based on place types

  • List View: Hierarchical tree view for organizing places with parent-child relationships

2. Place Types

Places are categorized by type with automatic color coding:

  • Country (Green), State (Orange), County (Yellow), City (Blue)

  • District (Purple), Building (Gray), Room (Teal)

  • Point of Interest (Red)

  • Custom types receive automatically generated colors

Getting Started

Accessing the Places Manager

  1. Navigate to the Places section in your application

  2. The Places Manager window will open with the Map View by default

Switching Between Views

  • Click "Map View" to see places on an interactive map

  • Click "List View" to see places in a hierarchical list

  • The active view button will be disabled (grayed out)

Using Map View

1. Viewing Places on the Map

  • Places with valid coordinates appear as colored markers

  • Click any marker to see place details

  • The map automatically zooms to show all visible places

2. Filtering by Place Type

  1. Click "Show Filters" to reveal the filter panel

  2. Use the multi-select dropdown to choose which place types to display

  3. Selected types will immediately appear on the map

  4. Click "Hide Filters" to collapse the panel

  5. Use "Refresh Types" to update the type list from the database

3. Interacting with Map Markers

  • Hover over marker: Shows place name as tooltip

  • Click marker: Opens detailed popup with:

    • Place name and type

    • Coordinates

    • Description (truncated if long)

    • "View Associations" button

4. Viewing Associations from Map

  1. Click any map marker

  2. In the popup, click "View Associations"

  3. A dialog will show all music tracks and other entities linked to that place

Using List View

1. Understanding the Hierarchy

  • Places are displayed in parent-child relationships

  • Drag and drop items to change parent-child relationships

  • Children are indented under their parents

  • Each entry shows:

    • Place name and type

    • Number of associations (linked entities)

    • Color coding for association status

2. Adding a New Place

  1. Click "Add Place" button

  2. Fill in the place details in the edit dialog

  3. Click "Save" - the place will appear in both views

3. Organizing Places

  • Drag and Drop: Click and drag any place to:

    • Make it a child of another place (drop onto parent)

    • Move it to root level (drop onto empty area)

    • Reorder siblings

  • Changes are automatically saved to the database

4. Context Menu Actions

Right-click any place in the list to access:

  • View Tracks: See all music tracks associated with this place

  • Edit: Modify place details (name, type, coordinates, description)

  • Merge: Combine this place with another (duplicate removal)

  • Split: Create separate places from this one

  • Delete: Remove the place (requires confirmation)

5. Viewing Details

  • Hover over any place: See detailed tooltip with full information

  • Double-click or use context menu: Open detailed view dialog

Working with Associations

What are Associations?

Associations link places to entities in your music library, such as:

  • Tracks

  • Albums

  • Artists

  • Playlists

Viewing Associations

You can view associations from:

  1. Map View: Click any marker → "View Associations"

  2. List View: Right-click any place → "View Tracks"

  3. Both methods open the same Association Details dialog

Association Information Displayed

  • Total number of associated entities

  • Breakdown by entity type

  • List of specific tracks/albums/artists

  • Options to navigate to associated items

Advanced Features

1. Dynamic Type Management

  • New place types are automatically detected

  • Custom types receive consistent, visually distinct colors

  • Type filter updates dynamically as you add new place types

2. Persistent Filtering

  • Map type filters are remembered during your session

  • Filters apply to both marker display and list view counts

  • Empty filter shows all places

3. Real-time Updates

  • Changes in one view immediately reflect in the other

  • Adding/editing/deleting places updates both views

  • Parent-child relationship changes update the hierarchy

4. Error Handling

  • Invalid coordinates show fallback message

  • Network issues show offline map state

  • Database errors provide clear feedback

  • Failed operations can be retried
